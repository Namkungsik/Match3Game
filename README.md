# ✨ 주요 기능

## Match-3 퍼즐 시스템

- 같은 색상의 보석을 3개 이상 연결하면 제거
- 가로 및 세로 방향 매칭 판정
- 연속 매칭(콤보) 처리

## 중력(Gravity) 시스템

- 보석이 제거된 공간으로 위에 있는 보석이 자연스럽게 아래로 이동
- 빈 공간에는 새로운 보석이 자동 생성

## 점수 시스템

- 보석 제거 시 점수 획득
- 콤보 발생 시 추가 점수 획득

## 제한 시간 시스템

- 제한 시간 동안 플레이 진행
- 매칭 성공 시 추가 시간이 지급되어 전략적인 플레이 가능

## 장애물 시스템

- 일반 보석과 구분되는 장애물 블록 구현
- 장애물을 고려한 퍼즐 플레이 가능

## UI 시스템

- 현재 점수
- 남은 시간
- 장애물 개수
- 게임 종료 화면

# 🧱 자료구조 활용

| 위치 | 자료구조 | 용도 |
|---|---|---|
| `Board` | 2차원 배열(`GamePiece[,]`) | 게임 보드 전체를 관리하며 각 좌표의 보석 상태를 저장 |
| `List<GamePiece>` | 연결된 보석 탐색 | 매칭된 보석을 저장하고 제거 대상 관리 |
| `Fill[]` | 열(Column) 관리 | 각 열의 낙하 상태와 채우기 과정을 관리 |

2차원 배열을 이용하여 보드 상태를 관리하고, 매칭된 보석은 `List`에 저장하여 한 번에 제거하도록 구현했습니다. 이후 중력 처리를 통해 빈 공간을 채우고 새로운 보석을 생성하는 흐름으로 게임을 구성했습니다.

# 💡 구현하며 해결한 문제

### 1. Match 판정 알고리즘

보석이 이동할 때마다 상하좌우 방향을 검사하여 같은 종류의 보석이 3개 이상 연결되었는지 확인하도록 구현했습니다.

### 2. 중력 처리

보석 제거 후 각 열을 순회하며 빈 공간을 확인하고, 위에 있는 보석을 아래로 이동시키는 방식으로 자연스러운 낙하를 구현했습니다.

### 3. 연속 매칭(Combo)

중력으로 새로운 보석이 내려온 뒤 다시 매칭이 발생하는 경우를 반복 검사하여 연속 콤보가 발생하도록 구현했습니다.

### 4. 제한 시간 보상

매칭 성공 시 제한 시간을 추가하여 단순히 빠르게 플레이하는 것이 아니라 전략적인 플레이가 가능하도록 설계했습니다.

### 5. 장애물 처리

일반 보석과 장애물을 구분하여 이동 및 제거 조건을 다르게 처리함으로써 다양한 퍼즐 패턴을 구현했습니다.
